# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please email security@example.com instead of using the public issue tracker. This allows us to address the issue before public disclosure.

Please include the following details in your report:
- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Suggested fix (if available)

## Security Updates

We take security seriously and will:
- Acknowledge receipt of your report within 48 hours
- Provide an estimated timeline for a fix
- Release security patches as soon as possible
- Credit you for the discovery (if desired)

## Supported Versions

Please see the project's release notes for information about which versions currently receive security updates.
